# Rewrite the agent runtime in Effect TS

We are rewriting `pi-ai`, `pi-agent-core`, and `pi-coding-agent` on top of Effect TS to gain structured cancellation through Fibers, typed errors at every boundary, dependency injection via `Layer`, `Schema`-based validation for tool args / model configs / persisted state, first-class concurrency primitives for parallel tool calls and retries, and built-in tracing/metrics for agent steps. The current code threads `AbortController` by hand, throws untyped errors, wires dependencies through constructors and module-level state, validates ad hoc with `typebox`, manages queues with raw Promises, and has no built-in observability — Effect addresses all six pain points directly. The cost is high (the three packages are ~75K LOC combined) and the gain only materialises if we commit to idiomatic Effect throughout, not as a plumbing layer beneath Promise-shaped code.
